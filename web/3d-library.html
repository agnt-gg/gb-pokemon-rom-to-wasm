Pokemon ROM -> 3D WASM console

A user-provided Game Boy ROM is decoded into Sharp SM83 machine-code blocks and lifted into a generated WebAssembly module on the server. The live 3D console runs the same browser system: WASM handles block/control-flow execution while a JS Game Boy runtime supplies MMU/MBC banking, PPU, OAM DMA, timers, joypad, APU, saves, and interpreter fallback for unsafe code.
