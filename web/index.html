Pokemon ROM -> browser-playable WASM system

A user-provided Game Boy ROM is decoded into Sharp SM83 machine-code blocks, then lifted into a generated WebAssembly module on the server. In this browser build, WASM drives the block/control-flow shell while a correctness-first JS Game Boy runtime supplies MMU/MBC banking, PPU, OAM DMA, timers, joypad, APU, saves, and interpreter fallback for unsafe code.
